## Why

テンプレート一覧が68件のフラットなリストとして存在しており、UIでの選択時に認知負荷が高い。地方区分・政令市などの意味的なグルーピングと順序を定義することで、テンプレートを構造的に整理し、ナビゲーションを改善する。

## What Changes

- `geo_template_groups` テーブルを新設し、グループの名前と表示順を管理する
- `geo_template_group_members` 中間テーブルを新設し、テンプレートとグループの多対多関係およびグループ内順序を管理する
- テンプレート一覧 API (`GET /api/templates`) のレスポンスにグループ情報を追加する（または新規エンドポイントを追加する）
- 地方区分グループ（8地方 + 日本全体）のシードデータを定義する

## Capabilities

### New Capabilities

- `geo-template-groups`: テンプレートグループの定義・管理。グループ間の順序、グループ内のテンプレート順序（都道府県 → その県の市区 → 次の都道府県）を保持する。1テンプレートが複数グループに所属可能（多対多）。

### Modified Capabilities

- `geo-bff-api`: テンプレート一覧取得 API にグループ構造を返す機能を追加する

## Impact

- DB: マイグレーションで2テーブル追加
- シードデータ: `seed-supabase.mjs` にグループ・メンバーデータを追加
- API: `GET /api/templates` のレスポンス仕様変更、または `GET /api/template-groups` 追加
- フロントエンド: テンプレート選択UIがグループ構造を受け取れるよう対応が必要
