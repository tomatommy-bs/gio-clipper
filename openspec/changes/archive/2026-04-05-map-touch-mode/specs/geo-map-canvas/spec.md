## ADDED Requirements

### Requirement: タッチ操作によるパン
マップキャンバスは1本指のタッチドラッグによるパン操作をサポートしなければならない（SHALL）。モードに関わらずタッチドラッグは常にパンとして動作する。

#### Scenario: タッチドラッグでパン（edit モード）
- **WHEN** edit モードでユーザーが地図上を1本指でドラッグする
- **THEN** マップキャンバスがドラッグ方向にスクロールする

#### Scenario: タッチドラッグでパン（pan モード）
- **WHEN** pan モードでユーザーが地図上を1本指でドラッグする
- **THEN** マップキャンバスがドラッグ方向にスクロールする

### Requirement: タッチタップによるエリア選択
edit モード時、タップ（ほぼ移動なしのタッチ）はエリア選択として扱われなければならない（SHALL）。

#### Scenario: edit モードでタップしてエリア選択
- **WHEN** edit モードでユーザーがエリア上をタップする（移動量4px未満）
- **THEN** そのエリアの写真割り当て編集画面が開く

#### Scenario: pan モードではタップしてもモーダルが開かない
- **WHEN** pan モードでユーザーがエリア上をタップする
- **THEN** 写真割り当て編集画面は開かない

## MODIFIED Requirements

### Requirement: エリアのクリック操作
ユーザーは edit モード時に限り、マップ上の任意のエリアをクリック（またはタップ）して写真割り当て操作を開始できなければならない（SHALL）。pan モード時はクリック/タップによるモーダル起動は行われない。

#### Scenario: 未訪問エリアをクリック（ドラッグなし・edit モード）
- **WHEN** edit モードでユーザーがドラッグを行わずに未訪問エリアをクリックする
- **THEN** そのエリアの写真割り当て編集画面（region-photo-clip）が開く

#### Scenario: 訪問済みエリアをクリック（ドラッグなし・edit モード）
- **WHEN** edit モードでユーザーがドラッグを行わずに訪問済みエリアをクリックする
- **THEN** そのエリアの写真割り当て編集画面が開き、既存の割り当てを変更できる

#### Scenario: ドラッグ後のマウスアップはクリックとして扱わない
- **WHEN** ユーザーがマップをドラッグした後にマウスボタンを離す
- **THEN** エリアの写真割り当て編集画面は開かない

#### Scenario: pan モードではクリックしてもモーダルが開かない
- **WHEN** pan モードでユーザーがエリアをクリックする
- **THEN** 写真割り当て編集画面は開かない
