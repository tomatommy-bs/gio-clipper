## Context

`GeoMapCanvas` はSVGで全エリアを描画しており、各エリアは `<g onClick>` でクリック可能。`GeoRegion` には既に `name` フィールドが存在する。現状、ホバー時の視覚的フィードバックはCSSのカラー変化のみで、エリア名の表示はない。

## Goals / Non-Goals

**Goals:**
- マップ上のエリアをホバーしたとき、エリア名をラベルとして表示する
- ホバー離脱でラベルを非表示にする
- 実装をシンプルに保ち、既存のトランスフォーム・クリック処理に影響しない

**Non-Goals:**
- タッチデバイスのホバー対応（タップ時のラベル表示は対象外）
- エリア名のカスタマイズ
- ツールチップの複雑なポジショニング（SVG外に出るケースの対応）

## Decisions

### SVG内テキストでラベルを描画する

**採用**: SVGの `<text>` 要素をエリアのbboxセンターに描画する。

**理由**: `GeoMapCanvas` 全体がSVGのtransformで移動・拡縮しているため、SVG内に描画することでズーム・パン時にラベルも自然に追従する。HTML絶対配置のツールチップはtransform座標からクライアント座標への変換が必要で複雑になる。

**代替案**: HTML側でツールチップを出す → SVGのtransformとの座標同期が複雑になるため不採用。

### ホバー状態はReact stateで管理する

`hoveredRegionId: string | null` をコンポーネントのstateとして持ち、各 `<g>` の `onMouseEnter` / `onMouseLeave` で更新する。

**理由**: ホバー状態はレンダリングに直接関与するため、refではなくstateが適切。

### フォントサイズはズーム倍率に反比例させる

SVG内テキストはtransformでスケールされるため、ズームアップ時に文字が巨大化する。フォントサイズに `1 / transform.scale` を乗じてビューポート上で常に一定サイズに見えるようにする。

## Risks / Trade-offs

- [エリアが小さい場合にラベルが他のエリアに重なる] → フォントサイズを小さめに設定し、`pointer-events: none` でインタラクションには影響しないようにする
- [ドラッグ中もhoverが発火する] → `isPanning` 中は `onMouseEnter` を無視することで対応
