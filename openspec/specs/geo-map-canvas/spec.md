## Requirements

### Requirement: マップキャンバスの表示
システムはテンプレート選択後、全エリアをSVGで描画したマップキャンバスを表示しなければならない（SHALL）。
未訪問エリアと訪問済みエリアは視覚的に区別されなければならない。

#### Scenario: 未訪問エリアの表示
- **WHEN** エリアに写真が割り当てられていない
- **THEN** そのエリアのSVGパスは破線のストロークで描画され、内部は透明またはプレースホルダー色で表示される

#### Scenario: 訪問済みエリアの表示
- **WHEN** エリアに写真が割り当てられている
- **THEN** そのエリアのSVGパスは、割り当てられた写真が土地の形状でクリップされて表示される

### Requirement: ズームとパン操作
マップキャンバスはズームイン・ズームアウトおよびパン（移動）操作をサポートしなければならない（SHALL）。
操作手段はホイールスクロール・ドラッグに加え、外部コンポーネントからの制御（コントロールパネル）も受け付けなければならない。

#### Scenario: ズームイン操作（ホイール）
- **WHEN** ユーザーがマップ上でズームイン操作を行う（ピンチまたはスクロール）
- **THEN** マップが拡大表示され、小さいエリアのSVGパスも視認・クリック可能になる

#### Scenario: ズームアウト操作（ホイール）
- **WHEN** ユーザーがズームアウト操作を行う
- **THEN** マップが縮小し、全体像が確認できる

#### Scenario: パン操作（ドラッグ）
- **WHEN** ユーザーがマップをドラッグする
- **THEN** マップキャンバスがスクロールし、表示エリアが移動する

#### Scenario: 外部コントロールからのズーム操作
- **WHEN** コントロールパネルのズームボタンが操作される
- **THEN** マップのズーム倍率がボタンに対応した値に変化する

#### Scenario: 外部コントロールからのパン操作
- **WHEN** コントロールパネルの矢印ボタンが操作される
- **THEN** マップが対応する方向にスクロールする

#### Scenario: 現在のズーム倍率の参照
- **WHEN** ズーム操作が行われる
- **THEN** 現在のズーム倍率が外部から参照可能な状態で更新される

### Requirement: 達成状況の表示
マップキャンバスは訪問済みエリア数と全エリア数を表示しなければならない（SHALL）。

#### Scenario: 達成状況カウンターの表示
- **WHEN** マップキャンバスが表示される
- **THEN** 「訪問済み N / 全体 M」の形式で達成状況が表示される

### Requirement: タッチ操作によるパン
マップキャンバスは1本指のタッチドラッグによるパン操作をサポートしなければならない（SHALL）。モードに関わらずタッチドラッグは常にパンとして動作する。

#### Scenario: タッチドラッグでパン（edit モード）
- **WHEN** edit モードでユーザーが地図上を1本指でドラッグする
- **THEN** マップキャンバスがドラッグ方向にスクロールする

#### Scenario: タッチドラッグでパン（pan モード）
- **WHEN** pan モードでユーザーが地図上を1本指でドラッグする
- **THEN** マップキャンバスがドラッグ方向にスクロールする

### Requirement: タッチタップによるエリア選択
edit モード時、タップ（ほぼ移動なしのタッチ）はエリア選択として扱われなければならない（SHALL）。

#### Scenario: edit モードでタップしてエリア選択
- **WHEN** edit モードでユーザーがエリア上をタップする（移動量4px未満）
- **THEN** そのエリアの写真割り当て編集画面が開く

#### Scenario: pan モードではタップしてもモーダルが開かない
- **WHEN** pan モードでユーザーがエリア上をタップする
- **THEN** 写真割り当て編集画面は開かない

### Requirement: エリアのクリック操作
ユーザーは edit モード時に限り、マップ上の任意のエリアをクリック（またはタップ）して写真割り当て操作を開始できなければならない（SHALL）。pan モード時はクリック/タップによるモーダル起動は行われない。

#### Scenario: 未訪問エリアをクリック（ドラッグなし・edit モード）
- **WHEN** edit モードでユーザーがドラッグを行わずに未訪問エリアをクリックする
- **THEN** そのエリアの写真割り当て編集画面（region-photo-clip）が開く

#### Scenario: 訪問済みエリアをクリック（ドラッグなし・edit モード）
- **WHEN** edit モードでユーザーがドラッグを行わずに訪問済みエリアをクリックする
- **THEN** そのエリアの写真割り当て編集画面が開き、既存の割り当てを変更できる

#### Scenario: ドラッグ後のマウスアップはクリックとして扱わない
- **WHEN** ユーザーがマップをドラッグした後にマウスボタンを離す
- **THEN** エリアの写真割り当て編集画面は開かない

#### Scenario: pan モードではクリックしてもモーダルが開かない
- **WHEN** pan モードでユーザーがエリアをクリックする
- **THEN** 写真割り当て編集画面は開かない
