# geo-travel-album

旅行写真を、訪れた土地の形状で切り抜いてコレクションとして並べるWebサービス。

---

## このプロダクトが解決する問題

旅行の記録は「写真フォルダ」や「SNS投稿」として残るが、どちらも**地理的な文脈**が失われる。「今年どこへ行ったか」「東北をどれだけ回ったか」「まだ行けていない県はどこか」—— こうした問いに答えるツールが存在しない。

また、単純に「旅行先の写真を土地の形にくり抜く」だけでも、それ自体がSNSで映える一枚になる。旅の記念として、次の旅へのモチベーションとして、2つの価値がある。

---

## コアコンセプト

### 1. 土地の形で写真をくり抜く

ユーザーは旅行先のエリア（例: 青森県）をクリックし、自分の旅行写真を選ぶ。写真は青森県の形状そのままでクリップされ、シルエット写真として表示される。

```
旅行写真 + 青森県の形 → 青森県シルエットの写真
```

この1枚だけでも、単体でダウンロードしてSNSでシェアできる。

### 2. 地図としてコレクションを育てる

訪れたエリアが増えるほど、地図が写真で埋まっていく。まだ行っていない場所は破線のプレースホルダーとして残り、「次はここへ行こう」という動機になる。

```
┌─────────────────────────────┐
│  青森 ██  岩手 ██  宮城 ╌╌ │
│  秋田 ╌╌  山形 ██  福島 ╌╌ │
│        写真あり=██          │
│        未訪問=╌╌ (破線)    │
└─────────────────────────────┘
```

---

## 主なユーザー体験

1. **テンプレートを選ぶ** — 「日本全国（都道府県）」「東京都（23区）」など
2. **マップが表示される** — 全エリアが破線で表示された空のマップ
3. **行ったエリアをクリック** — 写真を選んでクリップ位置を調整、保存
4. **地図が育っていく** — 訪問済みエリアに写真が入り、達成感が可視化される
5. **シェアする** — 地図全体・個別クリップをPNG画像としてダウンロード

---

## 提供形態

**Webアプリ（ブラウザで動作）**

- インストール不要、URLを開くだけで使える
- 写真データはすべてブラウザのローカルに保存（外部サーバーに送信しない）
- スマートフォンのブラウザでも動作する

## 将来フェーズ（MVPには含まない）

- **ネイティブアプリ**: iOSフォトライブラリと直接連携し、写真選択の手間をなくす
- **現地撮影フロー**: 旅先でその場でカメラ撮影 → 即座にクリップ登録
- **クラウド同期**: デバイスをまたいで同期、バックアップ
- **SNSシェア用URL**: 自分のコレクションをリンクで共有
- **地理的に忠実でないテンプレート**: 小さい県を見やすくした「デフォルメマップ」版
- **日本以外の地域**: 世界の国・地域への対応

---

## 技術方針（要点のみ）

- **フレームワーク**: Next.js（React）
- **地理データ**: 日本の行政区画GeoJSONをビルド時に前処理し、SVGパス文字列としてバンドル。離島（沖縄等）は天気予報と同様のオフセット表示
- **写真保存**: メタデータはlocalStorage、写真バイナリはIndexedDB（5MB制限回避）
- **書き出し**: Canvas API → PNG（長辺1024px以上・背景透過対応）

---

## このリポジトリについて

`shape-clipper.tsx` はブラウザ上でSVGクリッピングが動くことを検証したプロトタイプ。本実装はこのプロトタイプを破棄し、マップキャンバスを中心とした設計で新規に構築する。

詳細な要件・設計は `openspec/changes/geo-travel-album/` を参照。
